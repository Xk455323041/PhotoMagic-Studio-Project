## 🎯 交互流程设计

### 1. 单张图片处理流程
```
1. 用户进入页面
2. 拖拽或选择图片
3. 图片验证通过，显示缩略图
4. 调整处理参数（可选）
5. 点击"开始处理"
6. 显示处理进度
7. 处理完成，显示结果预览
8. 点击"下载PNG"保存结果
9. 可点击"重新开始"处理新图片
```

### 2. 批量处理流程
```
1. 点击"选择多张图片"
2. 选择多张图片（最多10张）
3. 设置批量处理参数
4. 点击"开始批量处理"
5. 显示批量处理进度
6. 处理完成，提供打包下载
7. 可继续添加图片或重新开始
```

### 3. 错误处理流程
```
1. 文件格式错误 → 显示错误提示，允许重新选择
2. 文件大小超限 → 提示压缩建议，提供在线压缩工具链接
3. 网络错误 → 自动重试，显示重试按钮
4. 处理失败 → 显示具体错误原因，提供解决方案
5. API限制 → 提示限制信息，建议升级或稍后重试
```

## 📱 响应式设计

### 断点适配策略

#### 移动端 (< 768px)
- **单列布局**: 所有区域垂直排列
- **简化控件**: 使用滑块代替单选按钮组
- **触摸优化**: 增大触摸目标
- **折叠面板**: 次要信息默认折叠

#### 平板端 (768px - 1023px)
- **双列布局**: 参数设置和预览区域并排
- **适中控件**: 平衡空间和可用性

- **多列布局**: 充分利用屏幕空间
- **丰富控件**: 显示所有选项

```typescript
  // 上传状态
  upload: {
    status: 'idle' | 'uploading' | 'uploaded' | 'error';
    file: File | null;
    previewUrl: string;
    error: string | null;
  };
  
  // 处理参数
  parameters: {
    format: 'png' | 'jpg' | 'webp';
    backgroundColor: 'transparent' | 'white' | string;
    edgeSmoothness: 'auto' | 'low' | 'medium' | 'high';
    hairDetail: boolean;
    shadow: boolean;
    quality: number;
    keepResolution: boolean;
  };
  
  // 处理状态
  processing: {
    status: 'idle' | 'processing' | 'completed' | 'error';
    progress: number;
    estimatedTime: number;
    resultUrl: string | null;
    error: string | null;
  };
  
  // 批量处理状态
  batch: {
    files: File[];
    status: 'idle' | 'processing' | 'completed';
    progress: Array<{ id: string; progress: number; status: string }>;
  };
}
```

### 状态流转
```
idle → uploading → uploaded → processing → completed
  ↓        ↓          ↓          ↓           ↓
重新开始  取消上传   调整参数   取消处理   下载结果
```
